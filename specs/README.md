# specs/

Feature specs live here, one directory per feature: `NNN-feature-name/` (e.g. `003-lightning-checkout/`).

For the company mission, product pillars, and scope boundaries, see [../docs/product-vision.md](../docs/product-vision.md). Feature specs should reference that document rather than re-litigating mission or payment stance.
